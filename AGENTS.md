# Codex Agent Rules

When assigned a new task or feature, **DO NOT directly output implementation code or modify files immediately**. You MUST strictly follow these 3 steps first:

## Step 1: Thought Process & Requirements

- Briefly summarize your understanding of my requirements.
- Explain the key architecture, recommended libraries/algorithms, and your reasoning.

## Step 2: Edge Cases & Clarifications

- List key technical details or parameters you need me to confirm.
- Identify potential edge cases, failure points, or error handling strategies for me to decide on.

## Step 3: Flowchart / Sequence Diagram

- Provide a clear Mermaid graph (`mermaid` code block) showing the execution flow and key decision paths.

---

**CRITICAL INSTRUCTION**:
Stop after completing Step 3 and wait for my response. Do NOT run code-editing tools or apply file diffs until I review and approve your plan.
